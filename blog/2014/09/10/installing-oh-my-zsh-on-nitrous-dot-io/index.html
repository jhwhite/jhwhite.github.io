Installing Oh-my-zsh on nitrous.io

Sep 10th, 2014 4:51 pm

I’ve started using nitrous.io for Rails development and learning after Rails School in Charlottesville VA started using it for their classes. The decision to use a cloud IDE was solidified after reading the draft chapters of Michael Hartl’s Rails tutorial where he advocates using Cloud 9.

On my local environment I use and have fallen in love with zsh with oh-my-zsh. So after using nitrous to write a test app I decided to see about installing oh-my-zsh and moving away from the bash prompt.

Your nitrous box will come installed with zsh.

At this point if you go back into your box and at the command line type rails new test-app you might/should get an error Couldn't find railties. It basically means that rails is broke, but this is easy to fix!

At the command line type gem install rails.

After the gem is finished installing let’s try to create our app again, rails new test-app. And your new app should be good to go!

Errors

It’s possible you will get an error that bundler couldn’t be found. Not a problem! At the command line run gem install bundler.

Wrap-up

oh-my-zsh is a great shell that is fun to use! Want to customize your install? You can find a list of preinstalled themes here: oh-my-zsh themes. None of those are to your liking? Make your own theme! Customize oh-my-zsh
